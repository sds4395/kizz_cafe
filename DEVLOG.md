- `CLAUDE.md`, `DEVLOG.md` 신규 작성 (두 PC 교대 작업 체계 도입).

### 추가 폼 개선 완료 (이번 세션)
- **이름 검색 → 자동채움**: 추가 폼을 카카오 장소검색(`searchPlaces`) 기반으로 재설계. 이름 검색 → 후보 선택 시 이름·주소·전화·좌표·지역 자동 입력, 사용자는 설명만 작성. E2E 검증 완료(서울형 키즈카페 강동구 성내1동점 실제 등록).
- **카드 선택 시 상단 스크롤**: `App.onSelect`에서 `window.scrollTo({top:0, behavior:'smooth'})`. 헤드리스 미리보기는 smooth 애니메이션 미지원이지만 실기기 정상.
- **시드 데이터 제거**: `seedCafes.ts` 삭제, `useCafes`가 Supabase 데이터만 사용.
- ⚠️ 시간당 요금은 카카오 로컬 API가 미제공 → 설명란에 수기 입력하도록 안내(placeholder).

### 현재 위치 기능 (이번 세션)
- **내 위치 기준 지도**: `App`에서 최초 진입 시 `navigator.geolocation`으로 현재 위치를 조용히 요청 → 지도 중심 = 검색 > 내 위치 > 기본(서울시청). 목록도 내 위치 기준 거리순 정렬·거리 표시.
- **현재 위치 버튼**: 지도 우하단 원형 버튼(십자 아이콘). 클릭 시 위치 재요청 후 재중심. 로딩 중 스피너, 권한 거부/실패 시 배너 안내.
- `MapView` 중심 이동을 `panTo`→`setCenter`로 변경(먼 위치로 이동 시 어색한 전국 애니메이션 방지). 선택 카페 이동은 `panTo` 유지.
- ⚠️ geolocation은 HTTPS(보안 컨텍스트)에서만 동작 → 배포(Vercel https)/localhost OK.

### 환경 메모
- 각 PC에서 `.env`는 직접 채워야 함(Git 미추적). 세 값: 카카오 JS 키, `https://sijtdpmslviphzwpowxx.supabase.co`, `sb_publishable_...` 키.
- ⚠️ Node를 새로 설치한 직후엔 실행 중이던 Claude Code/터미널 세션이 PATH를 못 물려받아 `npm`을 못 찾을 수 있음 → **앱/터미널 재시작**하면 해결.

### 데이터 씨딩 + 카카오 연동 (이번 세션)
- **공공데이터 씨딩**: 소상공인 상가정보 API(data.go.kr)로 강북/강동 키즈카페를 위경도 포함 조회 → 정제 후 Supabase에 7곳 삽입. (수집/삽입 스크립트는 scratchpad에만, 인증키는 커밋 안 함)
- **폐업 검증**: 브라우저 카카오 SDK로 각 카페를 검색해 좌표 대조 → 카카오에 없는 `키즈마루`, `봉봉키즈` 2곳은 폐업으로 확인(삭제 대상).
- **맵 오버레이 + ‘카카오로 보기’**: 카드 선택 시 지도에 이름 말풍선 + 버튼. 버튼은 `https://map.kakao.com/link/search/<이름>`(카카오맵 검색 화면)을 새 탭으로 염. 오버레이 텍스트는 XSS 방지 위해 textContent 사용.

### 추가 폼 개편: 카카오 검증 → 공공데이터 저장 (이번 세션, 코드 완료)
- **서버리스 프록시**: `api/place.js`(Vercel) + `vite.config.ts` dev 미들웨어가 공용 로직 `api/_findPlace.mjs` 사용. 좌표로 소상공인 반경조회(150m, numOfRows=1000) → 이름 유사도 매칭해 공공데이터 상호/주소/좌표 반환. 인증키 `SBIZ_API_KEY`는 서버측 전용(.env, VITE_ 아님 / Vercel 환경변수).
- **폼 로직**: 카카오로 검색·선택(실시간=존재확인) → 제출 시 `/api/place` 재조회 → 공공데이터로 치환(없으면 카카오 폴백) → 저장. 카카오 **장소ID/URL 저장**(약관 허용) → ‘카카오로 보기’ 장소상세 직링크.
- **검증 완료**: /api/place 프록시 정상(프레리키즈카페 치환 확인), 폼 제출 시 재조회→insert까지 흐름 확인. insert는 `kakao_id` 컬럼 필요(아래 SQL).
- ⚠️ **미완(사용자 SQL 필요)**: Supabase에 `kakao_id`,`kakao_url` 컬럼 추가 전엔 신규 추가가 400. 폐업 2곳 삭제/기존 카페 장소ID 백필도 같은 SQL로.
